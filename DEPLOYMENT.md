# Deployment Guide - Sudoku Multiplayer

This guide explains how to deploy the Sudoku Multiplayer frontend to **Vercel** and the backend server to **Railway**.

---

## Step 1: Push your Code to GitHub
Ensure all your local changes are committed and pushed to your GitHub repository:
```bash
git add .
git commit -m "Configure WebRTC improvements and socket proxy"
git push origin main
```

---

## Step 2: Deploy the Backend to Railway
1. Go to **[Railway.app](https://railway.app/)** and log in with your GitHub account.
2. Click **New Project** → **Deploy from GitHub repo** and select your Sudoku repository.
3. Once the service is created, click on it, go to the **Settings** tab.
4. Scroll down to the **General** section and find **Root Directory**.
5. Set the **Root Directory** to `server` (this tells Railway to only build and run the backend).
6. Railway will automatically deploy it. Once done, go to the **Variables** tab or the top of the page and copy the public URL generated by Railway (e.g., `https://sudoku-production.up.railway.app`).

---

## Step 3: Deploy the Frontend to Vercel
1. Go to **[Vercel.com](https://vercel.com/)** and log in with your GitHub account.
2. Click **Add New** → **Project** and select your Sudoku repository.
3. In the project configuration page, leave all build settings at their defaults.
4. Expand the **Environment Variables** section and add the following variable:
   * **Key:** `VITE_SOCKET_URL`
   * **Value:** `https://your-sudoku-backend.up.railway.app` (paste your Railway public URL here)
5. Click **Deploy**.

Vercel will compile the React frontend and link it to your active Railway Socket.io server. You're done!
